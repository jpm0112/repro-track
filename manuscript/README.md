# Manuscript sources

Submission paths for the two venues in the MLRC pipeline.

## `tmlr/`

TMLR / OpenReview submission. This is the primary submission — MLRC 2026
requires acceptance at TMLR before the compatibility review.

Pull the official TMLR LaTeX class from
[github.com/JmlrOrg/tmlr-style-file](https://github.com/JmlrOrg/tmlr-style-file)
and structure as:

```
tmlr/
├── article.tex          # \documentclass{tmlr} entry point
├── content.tex          # paper body
├── packages.tex         # imports
├── refs.bib             # citations
└── figures/             # plots, schematics
```

Anonymize before submission (TMLR is double-blind during review). A simple
sed-based stripper can be added under `reproduction/scripts/shell/` when ready.

## `rescience/`

ReScience C journal version, prepared *after* TMLR acceptance + MLRC
compatibility review per the editorial template at
[github.com/ReScience/MLRC](https://github.com/ReScience/MLRC).

The ReScience scaffold is generated by their `setup.sh <citekey>`; the result
includes:

```
rescience/
├── metadata.yaml        # title, authors, ORCID, code link, SWH ID
├── article.tex          # ReScience-formatted wrapper
├── content.tex          # body (often shared with tmlr/content.tex)
├── packages.tex
└── …
```

Code archival on
[Software Heritage](https://archive.softwareheritage.org/) is required before
submission. The resulting `swh:1:rev:<hash>` identifier goes into both
`metadata.yaml` and the paper body.
